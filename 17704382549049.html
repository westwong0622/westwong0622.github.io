直播第二天 - 尝试批量发布动态

2026/02/07

一、优化之前的笔记文章

问题：缩进后遗症

我尝试使用 媒体同步工具 进行多渠道发布，但同步后发现排版效果很差。

主要原因有两个：

  • 原始内容大量使用了 Markdown 缩进
  • 这是长期使用 Logseq 写笔记形成的习惯

另外还有一个问题：文本本身也需要进一步优化

临时解决方法

把 Markdown 内容直接发给 ChatGPT，让它帮忙优化：

  • 缩进压平
  • 表达优化

使用的提示词

{{note_text}}
---
角色：文章优化大师
需求：上面是我的 markdown 文档，采用的 logseq 写法。请改成普通缩进，以适配小红书、微信公众号的排版

新出现的问题

从 ChatGPT 回复中复制 Markdown 后：

  • 列表项之间会出现多余的空行
  • 尝试使用 Obsidian 的 Linter 无法解决
  • 当前只能 手动删除空行

这是一个明显值得工程化解决的问题，我会想办法解决

方案构思

  • 是否可以用 正则表达式 批量处理？
  • 是否可以写一个 小脚本 统一清洗？
  • 是否直接用 Claude Code 写一个进行清洗？

二、媒体运营工具体验

问了不少网友，也试用了几个工具。

真实感受只有一句话：好用的内容同步工具真的很难找。

目前唯一推荐

爱贝壳内容同步助手

Chrome 插件地址：
https://chromewebstore.google.com/detail/爱贝壳内容同步助手/jejejajkcbhejfiocemmddgbkdlhhngm

版本说明：

  • 免费版：每日 3 次发布、3 个平台
  • 付费版（不限）： https://copydog.art/publisher/vip
    • 价格：99 元 / 年，可拼车降低成本

爱贝壳内容同步助手使用流程

1. 选择内容形态

  • 动态：类似 Twitter / X，可配图，适合日常输出
  • 文章：长图文，偏博客型，建议整理后发布
  • 短视频（未测试）
  • 播客（未测试）

2. 选择发布渠道

  • 点击渠道右侧「查看」
  • 登录账号并授权
  • 插件即可自动填充内容

3. 发布动态

编辑内容：

  • 填写标题（部分平台有）
  • 填写正文
  • 添加图片或视频

发布流程：

  • 点击「开始同步」
  • Chrome 自动打开所选平台并填充内容
  • 人工检查后点击发送

建议：

  • 补充平台特有内容，如话题、圈子、标签
  • 高强度发布者可开启「直接发布」

4. 发布文章

发布前准备：

  • 使用 MWeb 生成静态网站
  • 发布到 GitHub Pages
  • 将文章 URL 导入爱贝壳

编辑阶段：

  • 填写标题、摘要、封面图
  • 开启 Markdown 模式
  • 图片使用原网页地址

问题依旧存在：

  • 各平台对 Markdown 缩进支持很差
  • 排版效果不可控

当前折中方案（但不满意）

  • 先用插件同步，等待内容填充
  • 优先修正公众号排版
  • 从公众号复制到其他平台
  • 若平台支持 Markdown，粘贴 Markdown 并手动改图链

结果：能用，但非常不优雅。

三、内容资产管理的新问题

内容如何管理？

使用什么工具存储文本和图片：

  • Apple Notes
  • Obsidian

发布状态如何管理？

  • 发布队列
    • 是否已发布
  • 发布在哪些平台

这还需要多实践尝试
